App for evaluating player's performace (Wireframe)

Player Evaluation App v 1.0

A concept app from a design challenge I attemped

Problem statement

A coach from the amateur football team “Blindside Beavers” is evaluating his athletes. The coach wants to input, share and display the evaluation to the athlete with his mobile app. Create a low-fidelity interaction flow that helps the coach and athlete to keep track of their development on their mobile app.

User requirements

The app should allow a user (coach):

  • View and search players on the team.
  • Input new evaluation based on last game performance. This includes adding any observations and improvement points.
  • Ability to share the evaluation to players via email or messaging app.
  • Able to view previous evaluation.
  • Able to view previous and upcoming team matches.

User onboarding

After user installed the app and opens first time, the onboarding screens will guide user how to use the app.

wireframe - Login screen

A Login screen - pretty standard looking. Nothing too fancy.

wireframe - Home screen

  • This is the first screen user will see after login.
  • Allows user to view, search and add players.
  • User is able to see his current players rating and an indication whether the ratings improved or got lower compared to last time.
  • User will be able to search player by name.

wireframe - Add Evaluation

  • User will be able to add new evaluation when clicking on a player from previous screen.
  • The circles are showing different evaluation metrics and their score for the player.
  • On tap on any circle, the user will be able to use slider to change the rating.
  • User will be able to add any observations.
  • User will be able to see last evaluation ratings for comparison.
  • The circles will be colored differently in real app for easier readability.

wireframe - Publish and Share

  • Upon clicking ‘publish’ on previous screen, the user will see this screen.
  • They can share this document with other people.
  • If a player is using the app, they will also see their profile for quick feedback.

Common general profiles are: specialist, twin skills, triple skills, well-rounder, multi-skilled.

wireframe - Team matches

  • User will be able to see upcoming and previous matches here.
  • Assumption - games status will be retrieved by some api and updated accordingly here.

Approach and constraints

  • The main goal was the app should be easier to add / edit players evaluation. I went with circles representing metrics because a visual chart is much better in understanding quantitative data.
  • Because of the mobile screen size, I went with 6 main metrics categories. It should be possible to add another screen which allows user to add any single category with more details.
  • Some screens are left out for assumption because of time constraints - add-user screen, add-game screen, signup screen, my-info screen.
