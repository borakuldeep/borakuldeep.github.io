App for managing employee vacations (Wireframe / UI screen)

Vacation Manager App v 1.0

App designed as an inhouse project in my previous company

Problem statement

For manager's it's hard to track vacation easily. They want a quick and easy solution for viewing and managing vacations.

User requirements

The app should allow a user (manager):

  • View all vacations for a given period of time (6 months)
  • Approve and reject vacations.
  • Ability to quickly view vacation overlaps and take actions.

The app should allow a user (employee):

  • View all their vacations for a given period of time (6 months)
  • Create a new vacation request.

wireframe - Home page

The home page should show easy summary of 6 month vacation status.

  • This is the first screen user will see after login.
  • Allows user to quickly get 6 months vacation status.
  • Use of colours allows user to see for which month action is needed.
  • Green - All good, no action needed.
  • Yellow - No overlapping but some vacations needs approval.
  • Red - Some upapproved vacations are overlapping with another.

wireframe - Monthly Page

  • This is the monthly page after user clicks on a month card.
  • Allows user to see vacation timeline for that month.
  • Allows user to filter by vacation status.
  • Use of colours allows user to see for which month action is needed.

wireframe - Action Page

  • This is a pop up when user clicks on vacation from previous page
  • Allows user to approve or reject a vacation
  • Allows user to add a note if needed.
  • Once action is taken, the home and monthly timeline will reflect the changes.

wireframe - Home Page: Employee

  • This is smilar to manager's view but employee can only view their vacations status.
  • Use of color is similar as previous pages.
  • The vacations are represented as circles here. Larger size means longer vacation.
  • User can open unapproved vacation and edit or delete it.
  • User can request a new vacation.

wireframe - Request new vacation

  • Finally a request form for vacation.
  • This is very simple form.
  • Upon submitting new request, the screens of manager and employee should be updated accordingly.
